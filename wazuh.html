Security Project • Monitoring

Wazuh Monitoring Lab

Built a Wazuh-based monitoring lab to practice endpoint visibility, log collection, authentication monitoring, alert review, and basic incident investigation workflows across Windows and Linux systems.

Wazuh SIEM Concepts Windows Logs Linux Logs Alert Review

Lab Goals

  • Collect logs from Windows and Linux lab systems.
  • Review authentication events and failed login activity.
  • Practice reading security alerts and understanding event context.
  • Build confidence using security monitoring tools in a controlled environment.

Skills Demonstrated

  • Basic SIEM workflow understanding
  • Endpoint monitoring concepts
  • Log review and alert triage
  • Windows and Linux event visibility
  • Security documentation

Example Investigation Workflow

  1. Identify an alert or suspicious authentication event.
  2. Review the affected host, username, source, and timestamp.
  3. Determine whether the activity was expected or unusual.
  4. Document the finding, possible cause, and recommended next step.

What I Learned

This project helped me understand how monitoring tools collect events, how alerts can be investigated, and why good documentation matters in security operations. It also helped connect my networking and systems knowledge to real SOC-style workflows.

Next Improvements

  • Add more Windows and Linux agents.
  • Create sample incident writeups for failed login activity.
  • Improve alert tuning to reduce noise.
  • Document common events and response steps.
