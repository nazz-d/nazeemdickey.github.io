Enterprise Homelab Architecture

Architected a virtualized homelab across rackmount and small form factor servers using Proxmox VE, a dedicated bare-metal OPNsense edge router, managed switching, Ubiquiti wireless, NAS storage, and secure remote access tooling.

Proxmox VE OPNsense Bare Metal Subnetting Headscale Netbird Docker LXC

Logical Network Diagram

This diagram shows the high-level lab design without exposing private IP addresses, public IPs, credentials, serial numbers, or device identifiers.

What I Built

  • Deployed OPNsense on dedicated hardware as the primary edge router and firewall.
  • Managed internal IP routing, DHCP, DNS, firewall rules, and ingress/egress traffic.
  • Built a Proxmox cluster-style lab across Dell R610, Dell R410, IBM M4, and Optiplex 5040 SFF hardware.
  • Hosted isolated LXC containers and Docker environments for internal services.
  • Integrated NAS storage, backup practices, switching, and Ubiquiti wireless access points.
  • Documented the physical and logical topology for support, troubleshooting, and future changes.

Skills Demonstrated

  • Bare-metal deployment and infrastructure planning
  • TCP/IP routing, subnetting, DHCP, and DNS administration
  • Firewall rule management and security-minded network design
  • Virtual machine, LXC container, and Docker service deployment
  • Encrypted remote access without exposing management ports publicly
  • Infrastructure documentation and troubleshooting discipline

Technologies Used

Tools, platforms, and hardware represented in the lab environment.

Operational Focus

  • Keep management surfaces behind controlled access paths.
  • Separate workloads so lab changes do not take down every service.
  • Document device roles, traffic flow, and service dependencies.
  • Practice recovery thinking through NAS storage and backup workflows.

Security Notes

I avoid exposing sensitive management ports directly to the public internet. Remote access is handled through encrypted access methods, segmented routing, and firewall rules. Public documentation is sanitized before publishing.
